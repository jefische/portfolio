I recently completed a comprehensive web accessibility audit and remediation project for a regional healthcare provider, evaluating a large WordPress-based website against the requirements of WCAG 2.1 Level AA. The project involved auditing representative page templates, global navigation components, forms, PDFs, third-party integrations, and dynamic content to identify barriers affecting users with disabilities.

Using a combination of automated and manual testing techniques, I assessed compliance with key accessibility criteria including keyboard navigation, color contrast, heading structure, form labeling, focus management, alternative text, semantic HTML, and screen reader compatibility. Testing was performed using industry-standard tools such as WAVE, axe DevTools, browser developer tools, and accessibility inspection utilities, alongside manual keyboard testing and screen reader testing with NVDA to evaluate the real-world user experience for individuals relying on assistive technologies.

A significant portion of the engagement focused on documenting findings, prioritizing remediation efforts by severity and user impact, and collaborating with stakeholders to implement practical solutions. The project also required evaluating third-party technologies, identifying vendor-controlled accessibility limitations, and providing risk-based recommendations for ongoing compliance.

Beyond accessibility testing, I analyzed site architecture, indexing behavior, XML sitemaps, and content management workflows within WordPress and the Avada ecosystem to better understand how accessibility issues could propagate across hundreds of pages. This included reviewing reusable components, navigation structures, Slider Revolution modules, and content publishing processes to determine the most effective remediation strategy.

Through this work, I strengthened my expertise in WCAG 2.1 AA conformance, accessibility auditing methodologies, assistive technology testing, semantic web development, and accessibility reporting. The project reinforced the importance of integrating accessibility into both technical implementation and organizational processes, helping ensure that digital experiences remain usable and inclusive for all users.
